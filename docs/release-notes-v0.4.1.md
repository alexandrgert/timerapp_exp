# TaskTimer link B24 — версия 0.4.1

## Что нового

### WebDAV при выходе — на выбор

В настройках WebDAV появился чекбокс **«При выходе только отправить локальную копию (без слияния с облаком)»**.

- **По умолчанию (выкл.)** — при выходе, как и раньше: сначала подтягиваются изменения с сервера и объединяются с локальной базой, затем отправка в облако.
- **Если включить** — при закрытии программы на сервер уходит **только ваша локальная** `data.json`, без слияния. Удобно, если вы уверены, что работали только на этом компьютере и не хотите, чтобы облако перезаписало локальные данные.

Если при выходе обнаружился **конфликт версий** (облако изменилось с прошлой синхронизации), при **следующем запуске** появится предупреждение — что именно было сделано при закрытии.

### Подробнее про объединение старых баз

Диалог **«Объединить базы старых версий»** стал информативнее:

- краткая сводка: сколько **новых задач**, **дополненных задач** и **сессий** появится;
- кнопка **«Показать подробности…»** — полный список изменений, в том числе отличия в настройках интерфейса.

### Надёжность и исправления

- **Ручное объединение:** ответ «Нет» в меню настроек больше **не блокирует** запрос при следующем запуске (отказ запоминается только при отказе на старте).
- **Повреждённый `data.json.bak`:** при битом резервном файле программа не падает — восстановление из `.bak` пропускается, используется основной файл или пустая база.
- **Завершение по сигналу ОС** (`SIGTERM`, `SIGINT`): сохранение и резервная копия выполняются безопаснее через очередь событий Qt.
- **Поиск старых баз:** каталог Qt больше не считается «legacy»-источником для автоматического предложения слияния.

## Кому подойдёт обновление

- Пользуетесь **WebDAV** и хотите контролировать, что происходит с базой **при выходе**.
- Обновляетесь с **0.4.0** — все улучшения 0.4.0 (legacy merge, защита базы, трей) сохранены.

Скачайте сборку для своей платформы на [странице релиза v0.4.1](https://github.com/alexandrgert/timer-app/releases/tag/v0.4.1).

| Платформа | Файл | Установка |
|-----------|------|-----------|
| **Linux** (amd64) | [`.deb`](https://github.com/alexandrgert/timer-app/releases/download/v0.4.1/tasktimer-link-b24-0.4.1-amd64.deb) | `sudo dpkg -i …deb && sudo apt-get install -f` |

### Linux

```bash
wget https://github.com/alexandrgert/timer-app/releases/download/v0.4.1/tasktimer-link-b24-0.4.1-amd64.deb
sudo dpkg -i tasktimer-link-b24-0.4.1-amd64.deb

tasktimer-link-b24
```

**Текущая база задач сохранится** при обновлении поверх установленной версии.

## На что обратить внимание

- Режим **«только отправить локальную копию»** при выходе может **перезаписать** более новую версию в облаке — включайте осознанно; при конфликте программа предупредит при следующем запуске.
- Защита базы при выходе **не гарантирована** при `kill -9` и внезапном отключении питания (как в 0.4.0).
- Подробности про WebDAV, legacy merge и трей — в [релизе 0.4.0](release-notes-v0.4.0.md).
